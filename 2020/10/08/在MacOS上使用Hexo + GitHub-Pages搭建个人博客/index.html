在MacOS上使用Hexo + GitHub-Pages搭建个人博客
2020-10-08|实践

  1. 介绍

    Hexo是基于NodeJs的静态博客框架，简单、轻量，其生成的静态网页可以托管在Github和Heroku上。

    GitHub-Pages是GitHub提供的一项静态页面托管功能，省去了自己买域名搭服务器的成本。

  5. 修改配置

    配置项保存在_config.yml中，常用配置包括：

    • title -> 网站标题
    • subtitle -> 网站副标题
    • description -> 网站描述
    • author -> 作者名字
    • language -> 网站使用的语言

    冒号后需要插入一个英文逗号，如title: y0suke

  6. 更换主题

    在Hexo主题官网上选择一个自己喜欢的主题，点击缩略图会跳转到demo页，点击标题会跳转到项目地址，参考官方给出的安装与配置教程进行操作，这里以melody主题作为示例。

    • 安装主题包与依赖

    • 找到_config.yml中theme那一项，修改为theme: melody

    • 复制melody主题的配置文件到博客根目录下，与_config.yml平级

    • 生成页面并重启服务

  7. 部署到Github-Pages

    • 创建GitHub账号

    • 创建一个名为[GitHub用户名].github.io的仓库，例如我的仓库名叫做y0suke-1006.github.io，权限保持为默认的public。

    • 安装部署插件

    • 修改_config.yml的deploy选项

    • 一键部署

      过一段时间后访问[GitHub用户名].github.io查看效果，例如我的博客地址y0suke-1006.github.io。
